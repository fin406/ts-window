Guide for AI agents and developers working on this project.

## Project Overview

A clean, professional marketing website for **TS Window Cleaning Services**, a Glasgow-based window cleaning and pressure washing business serving the West End and Southside.

### Tech Stack

| Layer | Technology |
|-------|------------|
| Framework | TanStack Start |
| Frontend | React 19, TanStack Router v1 |
| Build | Vite 7 |
| Styling | Tailwind CSS 4 |
| Icons | lucide-react |
| Forms | Netlify Forms |
| Language | TypeScript 5.7 (strict) |
| Deployment | Netlify |

## Brand & Design

- **Colours:** white background, brand blue (`--brand-blue: #1e60a8`), dark blue (`--brand-blue-dark: #154a82`), light blue tint (`--brand-blue-light: #e8f1fa`), neutral greys.
- **Layout:** content runs down the middle third — `.container-tight` (max 760px) for body content, `.container-wide` (max 1100px) for header/footer.
- **No photo backgrounds:** sections use solid colours, gradients, or light tints. Photos appear only as bordered cards within sections.
- **Stock imagery rule:** any photographic content must depict window cleaning or outdoor pressure washing — nothing else. The home page hero photo is the single stock image used; service descriptions use Lucide icons rather than photos to avoid mismatched imagery.

## Key Concepts

### File-Based Routing (TanStack Router)
Each `.tsx` file in `src/routes/` becomes a route. `__root.tsx` is the shared layout and renders an `<Outlet />` for nested pages. Route tree (`src/routeTree.gen.ts`) is auto-generated by the TanStack Router Vite plugin during dev/build.

### Netlify Forms
React renders forms client-side, so Netlify cannot detect them at build time. The static skeleton `public/__forms.html` registers each form (`contact`, `booking`) so submissions are accepted. React components POST URL-encoded form data to `/__forms.html` (NOT `/`, which would be intercepted by SSR). See `src/routes/contact.tsx` and `src/routes/book.tsx`.

When adding a new form:
1. Add a hidden `<form>` for it inside `public/__forms.html` with all field names.
2. POST from the React component to `/__forms.html` with matching `form-name`.
3. Run `node /opt/buildhome/.claude/skills/netlify-forms/scripts/enable.cjs` after changes.

## Conventions

### Naming
- Components: PascalCase
- Routes: kebab-case files
- Utilities: camelCase

### Styling
- Tailwind utility classes for everything
- CSS custom properties for brand tokens (`var(--brand-blue)` etc.)
- Arbitrary values via `[var(--brand-blue-light)]` pattern

### TypeScript
- Strict mode
- `@/` alias for `src/`
- Type-only imports with `type` keyword
